## 先搞清楚数据流

你用第三方入口跑 Codex 任务时，数据流向是这样的：

```
你的输入 → 第三方平台服务器 → OpenAI API → 模型处理 → 原路返回
```

这意味着你的代码/文件至少经过两个主体：第三方平台 + OpenAI。

---

## 三个核心风险

### 1. 第三方平台是否存储你的输入

关键问题：你的代码、文件、对话记录，平台会不会保存？保存多久？用不用于训练？

**怎么查**：找平台隐私政策/服务条款里的数据留存条款。没有隐私政策的平台，直接排除。

### 2. 传输过程是否加密

你的代码在网络上跑的时候是明文还是密文。

**怎么查**：看地址栏是不是 HTTPS，看平台有没有声明端到端加密。最基本的 HTTPS 都没有的站，别用。

### 3. OpenAI 侧的数据政策

通过 API 调用 OpenAI，根据 OpenAI 企业数据使用政策，API 输入默认不用于模型训练（截至 2026 年 6 月的官方说法）。但免费版 ChatGPT 的对话内容可能被用于训练。

**注意**：第三方平台走的是 API 通道还是免费账号通道，决定了你的数据是否参与训练。正规的 API 中转走的是 API，数据政策和企业版一致。

---

## 判断一个第三方入口靠不靠谱：3 个信号

| 信号 | 靠谱 | 不靠谱 |
| --- | --- | --- |
| 有没有隐私政策 | 有，且说明数据留存规则 | 没有，或只有一句"安全可靠" |
| 有没有企业主体 | 能查到备案/公司注册信息 | 查不到，或用个人站 |
| 走什么通道 | 明确说是 API 调用 | 含糊其辞，或用共享账号 |

---

## 实操建议

1. **核心代码别往任何第三方贴** — 不管平台多正规，公司核心业务代码都不该通过外部 API 跑。这是底线。
2. **测试性代码、学习代码随便用** — 非敏感代码、开源项目、练手项目，放心跑。
3. **看清楚走的是 API 还是共享账号** — API 通道数据政策清楚；共享账号通道等于把内容交给一个你不认识的账户主人。
4. **选有备案的平台** — 国内备案意味着有法律主体可追溯，出了事有人负责。
